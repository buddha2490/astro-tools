# astro-tools — Technical Overview

Audience: a Claude project instance picking this codebase up cold. Goal: be able to read, modify, and extend without re-discovering the structure.

## What this is

A collection of R scripts (plus shell/bat glue) that the owner uses to run an automated astrophotography rig and turn the raw output into something useful. It is **not a packaged R library** — it is a workshop of scripts invoked by hand, by `cron`, by Windows Task Scheduler, or by `R CMD BATCH` from shell wrappers. Many scripts assume specific working directories, drive mounts, and machine names.

The pipeline, end-to-end:

```
NINA capture rig (Windows mini-PC) ───► raw .fits + ImageMetaData.csv per session
                                           │
                              cleanup.R / postProcessing.R           (Windows — per night)
                                           │
                              bulkRename, processObjects, flat handoff to PixInsight WBPP
                                           │
                              transfer to NAS / Office-Mac SSD
                                           │
                              astroDB.R   (Mac — periodic inventory)
                                           │
                              SQLite db at /Volumes/Office-SSD/Astronomy/astroDB.sqlite
                                           │
                              queryastroDB.R  (Mac — interactive reporting via gt tables)

In parallel:
   NINA `.log` ──► NINALogAnalyzer (Shiny app) ──► Gantt + summary table
   PHD2 guidelog ──► copied into session folder by logFiles.R
```

## Runtime environment (load-bearing details)

### Machines and OS detection
Every entry-point script repeats this same `os`/`machine` detection block:

```r
os <- Sys.info()["sysname"]
machine <- Sys.info()["nodename"]
os <- ifelse(os == "Darwin", "Mac", "Windows")
machine <- ifelse(str_detect(machine, "BRIANC-MacUS"), "MBP13", ifelse(
           str_detect(machine, "Brians-MacBook-Pro"), "MBP14", ifelse(
           str_detect(machine, "Office-Mac"), "OfficeMac", ifelse(
           machine == "APERTURA-75" | machine == "MELE-ASTRO" | machine == "ES127", "TELESCOPE",
           machine))))
```

Machines referenced across the codebase:
- **MELE-ASTRO / APERTURA-75 / ES127** — Windows mini-PCs at the telescope (the "TELESCOPE" alias). Run NINA, PHD2, PixInsight, and the postprocessing/*.R scripts.
- **Office-Mac** — the desktop that mounts `/Volumes/Office-SSD/Astronomy/` and runs astroDB ETL.
- **MBP13 / MBP14** — Brian's laptops (dev).
- **Raspberry Pi at 100.85.227.75** — hosts a PostgreSQL `astroDB` referenced in `~/.Rprofile`'s `connectDB()` (see "Database split" below).

This duplicated detection block is technical debt — if you touch it, fix it everywhere. The `TELESCOPE` alias is the most important since most postprocessing paths are gated on it.

### Hard-coded paths (canonical)
- **Mac**: `/Volumes/Office-SSD/Astronomy/astro-tools` (repo), `/Volumes/Office-SSD/Astronomy/Astrophotography` (the canonical "subs" library), `/Volumes/Office-SSD/Astronomy/astroDB.sqlite` (DB).
- **Windows imaging PC**: `C:/users/bcart/Astronomy/astro-tools` (repo), `C:/users/bcart/astronomy/ASI2600MM/Subs` (active subs), `C:/Users/bcart/astronomy/ASI2600MM/Dark Library` (master darks/bias), `C:/Users/bcart/AppData/Local/NINA/Logs`, `C:/Users/bcart/Documents/PHD2`.
- **NAS transfer target**: `//100.67.219.30/ariabot/Astronomy/subs` (cleanup.R) or `//Office-Mac/Office-SSD/Astronomy/In Progress` (cleanup copy.R).

Most scripts `setwd()` to one of these. If you're modifying a script, preserve the `setwd()` line — downstream `source("functions/functions.R")` calls assume CWD.

### Database split (important and confusing)

There are TWO database connection functions named `connectDB()`:

| Location | Backend | Purpose |
|---|---|---|
| `.Rprofile` | PostgreSQL on Raspberry Pi (`100.85.227.75:5432`, db `astroDB`) | Auth via `Sys.getenv("username"/"password")` with `getPass` fallback. Loaded into the global env on R startup. |
| `astroDB/functions/functions.R` | SQLite at `/Volumes/Office-SSD/Astronomy/astroDB.sqlite` | Source()'d after `.Rprofile`, **overrides the Postgres one**. This is what `astroDB.R` and `queryastroDB.R` actually use. |

The `.Rprofile` Postgres `connectDB()` is dead code in current flows — the SQLite override wins. Leaving it alone is safe; just don't be misled by it when reading `.Rprofile`.

The SQLite DB has two tables:
- `astroSubs` — one row per `.fit` file, joined metadata + filesystem inventory
- `astroDBLog` — append-only audit log: `(Timestamp, Action, Status)`

## Module-by-module

### astroDB/ — image inventory + reporting

**`astroDB.R`** (the ETL): runs on the Mac.
1. `subsInventory()` scans `/Volumes/Office-SSD/Astronomy/Astrophotography/*/*.fit` (filesystem source of truth).
2. Finds all `ImageMetaData.csv` files recursively.
3. `changeMetaPath()` rewrites Windows-origin `FilePath` columns (`C:/Users/...`) to the NAS path, in place.
4. `processMetadata()` joins each row to the filesystem inventory by basename, classifies Status as `Included` / `Excluded` / `Missing Metadata`.
5. `fixMissingMeta()` recovers Object/Date from filename when metadata is absent.
6. Compares row count to existing `astroSubs` table; on diff, `dbWriteTable(... overwrite = TRUE)` and append to `astroDBLog`. **Full table overwrite, not incremental** — fine for current scale (~4,740 rows at dev) but worth knowing.

Filter encoding: filename suffixes `_L_`, `_R_`, `_G_`, `_B_`, `_H_`, `_S_`, `_O_`, `UVIR`, `_HO_` map to filter codes via a nested `ifelse` chain in `subsInventory()`. Duration is similarly parsed from `_30.00`, `_60.0`, `_120.0`, `_180.0`, `_300.0`, `_600.0` substrings. Both are open to refactor as named-vector lookups.

**`queryastroDB.R`** (interactive reporting). Source `functions/functions.R`, set `myObject <- "NGC281"`, then call:
- `checkLogs()` — most-recent row by Status from `astroDBLog`
- `astrobinCSV(myObject, csv = FALSE)` — produces an Astrobin-formatted sub tally (their numeric filter IDs are hardcoded as `c(4388, 4392, 4396, 5642, 5646, 5652, 5656, 6901, 13400)`)
- `objectTotalIntegration(myObject)` — `gt` table of subs/duration by filter
- `objectMetaData(myObject)` — guiding/HFR/FWHM/etc per filter as "mean (min-max)"
- `dbSummary()` — every object's total integration, LRGB vs Narrowband

All output is `gt`-rendered tables (not plain data frames) — these render in the RStudio Viewer.

### NINALogAnalyzer/ — Shiny app

`app.R` is a `shinydashboard` UI. User uploads a `.log` file → `pullLogs()` strips to timestamped lines → `eventPairs()` extracts `Starting Category:` / `Finishing Category:` pairs plus special-cased `Meridian Flip` → `cleanupLogs()` collapses related ROLEs into umbrella categories (`Slew, rotate, platesolve`, `Flats`) → `times()` computes durations → `logChartDev()` renders Gantt (ggplot → ggplotly) + summary table (DT).

**Quirks**:
- `allRoles`, `thisNight`, `starttime`, `endtime` are assigned with `<<-` (global). Don't refactor naively — downstream functions in `functions/functions.R` read from globals.
- Times are repeatedly re-cast through `as.POSIXlt` → `as.POSIXct` to force `America/New_York` tz. There's a `cleanTime()` helper but it's not used in the Shiny app.
- The Shiny app's `functions/functions.R` is a **partial copy** of postprocessing's. Same function names, different implementations in places. If you modify log-parsing logic, check both.
- Deployed to shinyapps.io (see `rsconnect/`).

### postprocessing/ — the Windows nightly flow

This is the most active and most fragile area. Runs on the TELESCOPE machine after NINA finishes a session.

**`postProcessing.R`** flow:
1. List directories in `c:/users/bcart/astronomy/ASI2600MM/Subs`, filter out non-session folders (`checkFits`, `flats`, `metadata`, `calibrated`, `logs`, `Flat_BIN`).
2. `bulkRename()` — strip spaces from `.fit` filenames (NINA sometimes leaves them).
3. `processObjects()` per object directory:
   - Rewrite `FilePath` in `ImageMetaData.csv` to use current `homePath`.
   - For each (CameraTargetTemp, Gain, Duration, FilterName) combo, compute z-score-style flags (`LowStars`, `HighHFR`, `HighFWHM`, `notRound`, `badGuiding`) on the metadata. Flagging logic is recorded but the move-to-`checkFits` step is **commented out** (replaced by manual SubframeSelector workflow — see `subframeSelector.R`).
   - `getDarks()` copies the matching master dark + master bias from the Dark Library into the session dir (matched by temp/gain/duration; temp clamped at `-20`).
   - `wbppFlats()` appends a line to `wbpp.bat` invoking PixInsight `WBPP.js` in automation mode for the flats.
4. After all objects are processed, run `wbpp.bat` via `sys::exec_wait()` (Windows only).

**`cleanup.R`** flow (runs after WBPP completes):
1. For each object, `cleanup()`:
   - `renameFlats()` — renames stacked master flats from PixInsight's verbose names (`*_L_mono.xisf` → `masterFlat_L.xisf`) and copies them up to the session dir.
   - Deletes `flats/calibrated`, `flats/logs`, `flats/fake`, and the whole `flats/` dir once masters are copied.
2. Clear and re-initialize `wbpp.bat`.
3. `source("logFiles.R")` to generate the night's PNG report.
4. Copy session directories to the NAS, then `unlink()` originals on local. Files (not dirs) are copied separately.

**`cleanup copy.R`** is `cleanup.R` with a different `transfer` destination — looks like a variant target for a different machine. Resolve duplication carefully; the two are 95% identical.

**`logFiles.R`** is invoked by `cleanup.R`. It:
1. Finds session folders by looking for `checkFits` subdirs (clever — these only exist after `processObjects` ran).
2. Locates the most recent NINA log in `C:/Users/bcart/AppData/Local/NINA/Logs`, archives older ones.
3. Locates the most recent PHD2 `GuideLog_*.txt`, copies into each session's `metadata` folder, deletes older guidelogs.
4. Parses the NINA log via the same `eventPairs` / `cleanupLogs` / `times` pipeline, joins with sub-level metadata via `addSubsToSequence()`, then `logChartDev()` produces a Gantt and a sub-tally table.
5. Combines into one PNG via `cowplot::plot_grid` and writes `{subsPath}/Imaging summary - {Sys.Date()}.png`.

**`functions/functions.R`** is the kitchen sink — `getDarks`, `processObjects`, `wbppFlats`, `bulkRename`, `cleanup`, `renameFlats`, `eventPairs`, `pullLogs`, `times`, `addSubsToSequence`, `reportGen`, `getMetaDataStartStop`, `getAllMetaData`, `processMetaData`, `cleanupLogs`, `cleanTime`, `getDarkTimesAPI` (sunrise-sunset.org for nautical twilight bounds, Atlanta hardcoded at `lat=33, lng=-84`), `logChartDev`. Note: also assigns `starttime`, `endtime`, `thisNight` with `<<-`.

### Top-level R scripts (one-shots)

- **`bulkRename.R`** — when you rename an object after capture (e.g., `M31StarCloud` → `NGC206`), edit `src`, `keyword`, `replacement` and run. Renames `.fit` files in place AND rewrites `FilePath` in `ImageMetaData.csv`.
- **`splitByFilter.R`** — explodes a session dir into per-filter subdirs so PixInsight BLINK can step through one filter at a time. After culling, copy survivors back and delete `L/R/G/B/H/O/S/` subdirs.
- **`subframeSelector.R`** — after running PixInsight SubframeSelector, you export `subframeSelector.csv`. This script reads it, computes 10%/90% quantile cutpoints per filter for SNR/FWHM/Eccentricity/Stars, and moves frames flagged as `Low SNR`/`Low FWHM`/`High Eccentricity`/`Low Stars` into an `Exclusions/` subdir.

### Shell wrappers
- `Shell/astroDB.sh` — what Mac cron calls nightly to run `astroDB/astroDB.R`. Output → `logs/astroDB.Rout`.
- `Shell/wbpp.sh` — Mac-side PixInsight FBPP automation against a hardcoded `M27` directory. Looks like a one-off; not part of the automated flow.
- `Shell/cronPI.sh` — references a `cronWBPP.R` that does not exist in the repo. Likely vestigial or held on the Pi.
- `postprocessing/postProcessing.bat`, `cleanup.bat` — Windows Task Scheduler entry points calling `R.exe CMD BATCH`.

## Cross-cutting patterns and gotchas

1. **Nested `ifelse` chains for filter/duration detection** appear in at least 5 places (`astroDB/functions`, `splitByFilter.R`, `subframeSelector.R`, `postprocessing/functions`). They are not identical — `subframeSelector.R` returns `"Problem"` as fallback, `splitByFilter.R` returns `"Unknown"`, `astroDB` returns `""`. If you unify them, audit each call site.

2. **`<<-` global assignment** is used in `NINALogAnalyzer/functions/functions.R` and `postprocessing/functions/functions.R` for `allRoles`, `thisNight`, `starttime`, `endtime`, `logFilePath`. The downstream functions silently depend on these globals. Refactoring to pass them as args is desirable but is a multi-file change.

3. **`%>%` (magrittr) and `library(dplyr)` everywhere** — not native `|>`. Match the existing style.

4. **No tests.** `testthat` is loaded in `.Rprofile` but there are no test files.

5. **No CI / GitHub Actions.** Local-only workflow.

6. **`.gitignore`** covers only `.Rproj.user`, `.Rhistory`, `.RData`, `.Ruserdata`, `*.Rout`, `/logs`. The `data/` directory IS checked in (sample logs, horizon files, camera JSON, sample reports).

7. **Time zones** — almost everything is `America/New_York`. NINA logs are UTC; conversion happens in `times()` and `cleanTime()`. Don't lose this when touching log code.

8. **PixInsight integration is shell-out only.** WBPP is invoked by writing a `.bat` file that's then executed (`sys::exec_wait()` on Windows, a direct shell call on Mac). The R side never talks to PixInsight via API.

9. **No package management** (`renv`, `packrat`). `.Rprofile` `library()`s a fixed set: `dplyr`, `magrittr`, `parallel`, `openxlsx`, `DBI`, `glue`, `readr`, `testthat`, `utils`, `getPass`, `RPostgreSQL`. Individual scripts load more (`shiny`, `shinydashboard`, `plotly`, `DT`, `lubridate`, `tidyr`, `ggplot2`, `gt`, `cowplot`, `patchwork`, `gridExtra`, `ggplotify`, `forcats`, `scales`, `RSQLite`, `httr`, `jsonlite`, `gtools`, `sys`).

10. **Branch state**: working on `main`. Recent commits show iteration on a "starfront" feature and consolidation of postprocessing for the mini-computer rig.

## Suggested entry points when picking up work

- "Where do I add a new object query?" → `astroDB/functions/functions.R`, follow the pattern of `objectMetaData()`. Then expose in `queryastroDB.R`.
- "Why is the nightly PNG broken?" → start in `postprocessing/logFiles.R`, then trace into `postprocessing/functions/functions.R` (`eventPairs` → `cleanupLogs` → `times` → `logChartDev`). Compare against `NINALogAnalyzer/functions/functions.R` which sometimes drifts.
- "Adding a new filter or sub-duration" → update the `ifelse` chains in `astroDB/functions/functions.R::subsInventory()` and `subframeSelector.R::getFilter()` and `splitByFilter.R::splitFilter()`.
- "Changing the rig" → update the machine-detection block in every entry-point script (`.R`, not the `functions/` files).
- "Switching DB backend" → flip `astroDB/functions/functions.R::connectDB()`. Confirm `astroDBLog` and `astroSubs` schemas match.
